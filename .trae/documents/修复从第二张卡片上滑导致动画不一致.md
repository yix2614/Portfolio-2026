## 问题判断
- 当前有两层动画同时存在：外层容器 Hero↔Card 的变形动画 + 内层卡片切换的淡入淡出。
- 当你在第 2 张卡片持续上滑时，会先触发“卡片切换动画”，再快速触发“回到 Hero 变形动画”，两套动画重叠，导致视觉上不是严格倒放。
- 这不是 UI 样式冲突，而是输入事件在动画未完成时被多次接受，导致状态顺序被打断。

## 修复方案（不改 UI 位置）
1. 添加统一的动画锁（isTransitioning ref/state），在动画执行期间拒绝新的滚轮/拖拽触发。
2. 为内层卡片切换的 AnimatePresence 增加 onExitComplete 回调，在切换结束后释放锁。
3. 为外层容器变形动画增加 onAnimationComplete 回调，在 Hero↔Card 变形完成后释放锁。
4. 在 handleNext/handlePrev 中严格判断：当 currentIndex>0 只允许切卡；只有 currentIndex==0 且动画结束后才允许切回 Hero。

## 验证方式
- 从第 2 张卡片连续上滑：必须先完整回到第 1 张，再完整回 Hero，动画倒放一致。
- 从 Hero 下滑进卡片再继续下滑：前进路径和回退路径一致且不会跳帧。

确认后我将只调整动画锁与状态顺序逻辑，不动任何 UI 布局或样式。
