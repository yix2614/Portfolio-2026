# 修复 ASCII 卡片上光标未隐藏的问题

## 1. 问题诊断
*   **现象**: 在 Card 模式下，鼠标悬停在 ASCII 卡片上时，系统默认箭头光标依然可见，尽管我们在父级 `CardWrapper` 上设置了 `cursor: none`。
*   **原因**: `AsciiScene` 内部由 Three.js 的 `AsciiEffect` 生成了一个 `canvas` (或 table/pre) 元素。这些生成的 DOM 元素可能会继承浏览器的默认用户代理样式（User Agent Styles），或者 Three.js/AsciiEffect 内部可能隐式应用了某些样式。更常见的是，如果子元素没有显式继承（`inherit`）或覆盖，它可能会回退到默认的 `auto` 或 `text` 光标（特别是如果它是基于文本的 ASCII）。
*   **验证**: 之前只检查了 React 代码中的样式，但 `effect.domElement` 是动态插入的，它的样式可能没被完全覆盖。

## 2. 解决方案
强制在 `AsciiScene` 组件内部对生成的 DOM 元素应用 `cursor: none` (当处于 Card 模式时)，或者简单地在所有模式下让其继承父级光标设置。

### 2.1 修改 `AsciiScene` 组件
*   在 `AsciiScene` 组件的 `useEffect` 中，当配置 `effect.domElement` 的样式时，显式添加 `cursor: 'inherit'` 或 `pointer-events: 'none'`。
    *   **注意**: 如果设置 `pointer-events: none`，则内部的三维交互（虽然我们之前删除了，但如果未来加回）会失效。但鉴于目前交互由 `CardWrapper` 接管，设为 `none` 是安全的且能穿透到下层。
    *   **最佳实践**: 给 `effect.domElement.style.cursor = 'inherit'`。这样它就会自动采用父级 `CardWrapper` 的 `none` 或 `default` 设置。

### 2.2 双重保险：在 `CardWrapper` 中强制全局样式
*   虽然不推荐，但也可以在 `CardWrapper` 的 CSS 中使用 `* { cursor: inherit !important }`，但这太暴力。
*   首选方案是在 `AsciiScene` 的容器 `div` 上也加上 `cursor: inherit`，并确保 `effect.domElement` 也继承。

## 3. 实施步骤
1.  **修改 `AsciiScene` (VibeCodingPage.tsx)**:
    *   在 `AsciiScene` 的返回 JSX 中，给容器 `div` 添加 `cursor: 'inherit'`。
    *   在 `useEffect` 中，找到配置 `effect.domElement.style` 的地方，添加 `effect.domElement.style.cursor = 'inherit'`。
    *   或者，更彻底地，给 `effect.domElement` 添加 `pointer-events: none`（如果不需要其内部事件），这样鼠标事件直接由父级 `CardWrapper` 捕获，自然使用父级光标。考虑到我们之前移除了内部交互，这是最干净的方法。

## 4. 预期结果
*   悬停在 ASCII 卡片上时，系统箭头光标消失，仅显示自定义的 "OPEN" 圆形光标。
*   交互（点击打开链接）依然有效（由父级 `CardWrapper` 处理）。
