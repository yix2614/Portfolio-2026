# 修复 ASCII 卡片高度塌陷计划

## 1. 问题根源
*   **高度塌陷**: 之前的修改将 Card 模式下的容器高度改为了 `auto`。
*   **AsciiScene 行为**: `AsciiScene` 内部是一个空的 `div` (挂载点)，Three.js 的 Canvas 是绝对定位插入的（或者即使是相对定位，初始也是空的）。如果不显式指定高度或宽高比，`height: 100%` 在父级为 `auto` 时会计算为 0。
*   **对比**: `Card3D` 包含 `<video>` 元素，它具有固有尺寸（Intrinsic Size），因此可以撑开 `auto` 高度的父容器。

## 2. 解决方案
为了让 `AsciiScene` 在 Card 模式（`height: auto`）下有高度，必须给它施加一个宽高比约束。

### 2.1 修改 `AsciiScene` 组件
*   **增加 aspect-ratio**: 在 `AsciiScene` 的根 `div` 上添加 `aspectRatio: '16 / 9'`。
*   **兼容性**:
    *   在 **Hero 模式** 下，父容器强制高度 `85vh`，`height: 100%` 优先级较高（或结合 `min-height`），通常 Flex/Grid 布局下会有特定表现。最稳妥的是仅在 Card 模式或通用情况下应用 `aspect-ratio`，但 Hero 模式通常会忽略它如果高度已定。
    *   更好的方式是：在 `AsciiScene` 内部检测或接收 `isHeroMode` prop，或者简单地加上 `aspect-ratio: 16/9`，因为 Hero 模式的强制高度会覆盖它（只要 `height` 属性是显式的）。

### 2.2 备选方案：修改 `CardWrapper`
*   由于 `CardWrapper` 已经包裹了 `AsciiScene`，可以在 `CardWrapper` 层面处理。
*   如果 `children` 是 `AsciiScene`（或者通用地），在 Card 模式下给 Wrapper 内部容器加上 `aspect-ratio: 16/9`。
*   **注意**: 这样会强制所有卡片都是 16:9。但用户之前要求的是“自适应内容高度”。
    *   对于 `Card3D`（视频），`auto` 高度工作正常（自适应视频比例）。
    *   对于 `AsciiScene`（Canvas），没有内容撑开，必须强制给一个比例。

### 2.3 最终方案
**仅针对 `AsciiScene` 的容器添加默认比例支撑。**
修改 `AsciiScene` 组件的 `style`：
```typescript
style={{ 
    width: '100%', 
    height: '100%', 
    // 新增：确保在父级 auto 时有高度。
    // 在 Hero 模式下，父级固定高度会覆盖此效果（或共同作用）。
    // 但为了保险，我们可以设为 min-height 或 aspect-ratio。
    aspectRatio: '16/9' 
}} 
```
这样，当父容器变为 `auto` 时，`AsciiScene` 会占据 16:9 的空间；当父容器是 `85vh` 时，它会拉伸以填满（`height: 100%` 生效）。

## 3. 实施步骤
1.  **修改 `AsciiScene` 组件**:
    *   找到 `VibeCodingPage.tsx` 中的 `AsciiScene` 组件定义。
    *   在返回的 `div` style 中添加 `aspectRatio: '16/9'`。
2.  **验证**:
    *   检查 Hero 模式：确认是否依然铺满 85vh（不应受 16:9 限制而留白）。
    *   检查 Card 模式：确认 ASCII 卡片是否恢复显示，且比例正确。

## 4. 预期结果
*   Hero 模式：ASCII 视频全屏展示，高度 85vh。
*   Card 模式：ASCII 卡片以 16:9 比例显示，不再塌陷消失。
