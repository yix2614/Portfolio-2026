## 结论
可以做到 1:1，但必须以当前“可用且一致”的 UI 为基线，然后逐步替换依赖，每一步都做视觉对比，不允许偏差。

## 目标
- UI/交互与当前 Framer 版本保持完全一致
- 外链与资源 URL 不变
- 拆分组件不改变任何样式/结构输出

## 计划
1. 建立对照基线
   - 确认当前版本是“正确 UI”并截图/记录关键状态（hover/展开/视频）
2. 拆分但不改渲染结果
   - 保持 ProjectMenu 渲染输出不变，仅做文件拆分与导入路径调整
3. 自研替换路线（逐步）
   - 先替换样式依赖（复制 unframer CSS 到本地）
   - 再替换运行时中的无副作用部分（Link/RichText/Image）
   - 最后替换变体/动画/布局逻辑（motion/variants/layout）
4. 每步验证
   - 每次替换后进行 UI 对比，若不一致立即回退并定位差异

我会严格按“基线 → 小步替换 → 逐步验证”的顺序执行，保证 1:1。
