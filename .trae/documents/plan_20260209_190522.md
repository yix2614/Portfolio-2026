# Vibe Coding 页面重构计划：全屏滚动与 Hero 变形交互

## 1. 核心目标
将现有的分层布局重构为**单页全屏滚动 (Whole Page Scroll)** 体验。实现 Hero Section（ASCII 视觉）与后续 Card Section（3D 视频卡片）的无缝融合。

## 2. 交互逻辑设计
引入双重状态管理：`isHeroMode` (Hero 模式) 和 `currentIndex` (卡片索引)。

*   **初始状态**: 用户进入页面，处于 `isHeroMode = true`, `currentIndex = 0`。
    *   **视觉**: 显示全屏 ASCII Hero，"VIBE CODING" 文字可见。
*   **向下滑动 (Step 1)**:
    *   **动作**: 触发从 Hero 模式进入 Card 模式。
    *   **动画**:
        *   Hero 容器从全屏尺寸 (`85vh`) 平滑缩小至卡片尺寸。
        *   "VIBE CODING" 文字淡出。
        *   Card 信息的顶部导航 (Title/Subtitle) 淡入。
        *   **内容**: 依然是 ASCII 场景，内容不切换，仅容器变形。
    *   **状态**: `isHeroMode = false`, `currentIndex = 0`。
*   **继续向下滑动 (Step 2)**:
    *   **动作**: 切换到下一张卡片。
    *   **动画**: 卡片内容切换（从 ASCII 变为普通视频卡片），带有位移/淡入淡出效果。
    *   **状态**: `isHeroMode = false`, `currentIndex = 1`。
*   **向上滑动**: 执行相反逻辑。

## 3. 技术实现方案

### 3.1 数据结构改造
更新 `slides` 数组，将现有的 Hero 视频作为第一个条目加入，并区分渲染类型：
*   **Slide 0 (原 Hero)**: `type: 'ascii'`, 视频源: `jimeng_gidwco.mp4`。
*   **Slide 1+ (原 Cards)**: `type: 'video'`, 保持现有视频。

### 3.2 组件架构重构
*   **主容器**: 设置 `overflow: hidden`，禁用原生滚动条，通过 `onWheel` 和 `useDrag` 接管滚动交互。
*   **统一视觉层 (Visual Layer)**:
    *   使用 `framer-motion` 的 `motion.div` 作为核心容器。
    *   定义 `variants` 处理两种形态：
        *   `hero`: 宽度 100%, 高度 85vh, 顶部对齐。
        *   `card`: 宽度 86vw, 高度自适应 (约 50vh), 居中对齐。
    *   **内容渲染**: 根据 `currentSlide.type` 动态渲染 `<AsciiScene>` 或 `<Card3D>`。
*   **文字层 (Text Layer)**:
    *   分离 "VIBE CODING" 大标题与 Top Nav 信息。
    *   根据 `isHeroMode` 状态交替显示（Fade In/Out）。

### 3.3 动画细节优化
*   **无缝变形**: 确保 `AsciiScene` 在容器尺寸变化时自动重绘 (Resize Observer)，保持画面比例。
*   **丝滑过渡**: 使用 `spring` 动画曲线，确保缩小和放大的物理质感。
*   **滚动阻尼**: 在模式切换和卡片切换之间添加适当的时间锁 (Throttle)，防止误触。

## 4. 实施步骤
1.  **数据准备**: 修改 `slides` 常量，合并 Hero 内容。
2.  **布局调整**: 移除 `secondSection` 独立区块，建立统一的 `Fixed` 布局结构。
3.  **状态逻辑**: 实现 `handleWheel` 的多级判断逻辑 (Hero Mode <-> Card Mode <-> Slide Change)。
4.  **动效开发**: 编写 `motion.div` 的变形动画和文字层的进出场动画。
5.  **验证**: 测试滚动流畅度、尺寸适配及 ASCII 效果的保留与销毁。
