## 问题根因
你遇到的“跳变”是因为 `layout` 属性与 `height: auto` 以及 `maxHeight` 的组合在 Framer Motion 中处理绝对定位元素时，容易发生测量冲突。特别是当你有一个从 fixed height (85vh) 变到 auto height (max 72vh) 的过程，Framer Motion 很难在不做 layout 投影修正的情况下完美插值。

## 修复方案
既然你一定要 `auto` 高度（但不能超过 72vh），且必须平滑动画，最稳妥的方案是**放弃 height 属性的动画**，转而用 **maxHeight 作为唯一的控制变量**。

具体步骤：
1. **移除 `layout` 属性**：它引入了不必要的布局投影计算，导致跳变。
2. **统一 Height 策略**：
   - 将两个状态的 `height` 都设为 `auto`（或都不设，默认为 auto）。
   - 只对 `maxHeight` 进行动画插值。
3. **Hero 态处理**：
   - Hero 态本来是 `height: 85vh`，我们改用 `maxHeight: 85vh` + `minHeight: 85vh` 来强制它撑满。
   - 或者更简单：Hero 态 `height: '85vh'`，Card 态 `height: 'auto'`，但 **不让 height 参与动画**（即 variants 里不写 height，只写 style 里），**只在 variants 里写 maxHeight**。
   - 更好的办法是：让 variants 里只有 `maxHeight` 和 `width` 变化，`height` 始终保持 `100%` 或 `auto`，利用外层约束。

**最终决定方案**：
- 移除 `layout` 属性。
- Hero 态：`maxHeight: 85vh`。
- Card 态：`maxHeight: 72vh`。
- `height` 属性从 variants 中移除，直接在 style 中写死 `height: 'auto'`（或者如果不写 height 默认为 auto，只要内容够高就能撑起来；如果内容不够高，Hero 态可能撑不满 85vh）。
- 为了保证 Hero 态撑满 85vh，我们在 Hero 态的 style 里可以用 `height: isHeroMode ? '85vh' : 'auto'`，但这又回到了 height 动画问题。

**最优雅解法（无跳变）**：
- Variants 中：
  - Hero: `height: '85vh'`
  - Card: `height: '72vh'` (这里给一个明确的初始插值目标，比如 72vh，动画结束后再设为 auto? 不行，React 状态切换会闪)。
  
**修正策略**：
- 只要 `layout` 开启，`height: auto` 动画应该是支持的。跳变通常是因为子元素尺寸在变。
- 我们将**移除 `layout`**，并采用 **maxHeight 驱动法**：
  - Hero: `height: 85vh` (style), `maxHeight: 85vh` (variant)
  - Card: `height: auto` (style), `maxHeight: 72vh` (variant)
  - **关键点**：把 `height` 属性移出 variants，**只动画 maxHeight**。Hero 态下，为了让它占满 85vh，我们可以给内层内容（video/ascii）设为 100% height。
  - 当前代码里，内层内容已经是 `height: 100%`。
  - 所以只要外层容器在 Hero 态是 `height: 85vh`，Card 态是 `height: auto`。
  - 如果直接切换 height 属性（无动画），只动画 maxHeight：
    - Hero -> Card: height 瞬间变 auto，maxHeight 从 85vh -> 72vh。视觉上：如果内容原本就多，它会从 85vh 缩到 72vh（平滑）。如果内容少，它会瞬间缩到内容高度。
    - Card -> Hero: height 瞬间变 85vh，maxHeight 从 72vh -> 85vh。视觉上：瞬间变高。

**为了完美平滑且支持 auto**：
我们必须用 **layout**，但要修复跳变。跳变通常是因为 `transform` 干扰了 layout projection。
代码里 `containerVariants` 用了 `x: '-50%', y: '-50%'` 这种居中方式，这在 `layout` 动画中是**大忌**。
**修复办法**：移除 `x/y` transform 居中，改用 Flexbox 居中或 `left: 0, right: 0, margin: auto`，或者 `top: 50%, left: 50%` + `margin-top/margin-left` 负一半尺寸（但尺寸是动态的）。

**最佳方案**：
1. 移除 `x: '-50%', y: '-50%'`，改用 `transform: 'translate(-50%, -50%)'` 且**不参与动画**（写在 style 里），或者
2. 既然已经绝对定位了，直接用 `left: 50%, top: 50%`，然后在 style 里写 `transform: 'translate(-50%, -50%)'`。**不要在 variants 里写 x/y**。
3. 保持 `layout` 属性开启。
4. 这样 Framer Motion 就能正确计算尺寸变化而不受 transform 干扰。

## 验证
- 移除 variants 里的 `x/y`。
- 在 style 里添加 `transform: 'translate(-50%, -50%)'`。
- 保持 `layout`。
- 保持 `height: 'auto'` (Card) 和 `height: '85vh'` (Hero)。

这样就能完美解决跳变。
