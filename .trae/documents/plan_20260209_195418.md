# 光标跟随与显隐逻辑修复计划

## 1. 问题诊断
*   **位置偏移**: 光标使用了 `position: fixed`，但被包裹在应用了 `transform` 动画的父容器中，导致定位基准错误（不再是视口，而是父容器）。
*   **显隐逻辑缺陷**: 当前逻辑虽然有 `isHovering` 状态，但由于上述定位问题，导致视觉上看起来“乱跑”或在卡片外也能看到（实际上可能是位置计算错了）。且用户希望只有 hover 到卡片时才显示，而非仅仅是缩放。
*   **性能隐患**: 使用 React State (`useState`) 追踪高频的 `mousemove` 事件会导致严重的性能问题（每秒重渲染 60+ 次）。

## 2. 解决方案
重构 `CardWrapper` 组件的光标实现逻辑。

### 2.1 引入 React Portal
*   使用 `createPortal` 将光标直接挂载到 `document.body`。这将彻底解决 CSS `transform` 导致的层叠上下文（Stacking Context）陷阱，确保 `fixed` 定位永远相对于浏览器视口。

### 2.2 优化性能 (Framer Motion)
*   引入 `useMotionValue` 替代 `useState` 来追踪鼠标坐标（`cursorX`, `cursorY`）。
*   这样可以直接驱动 DOM 样式更新，完全绕过 React 的 Render Cycle，实现 60fps 的丝滑跟随。

### 2.3 修正显隐逻辑
*   **状态控制**: 保持 `isHovering` 状态，由 `onMouseEnter` 和 `onMouseLeave` 触发。
*   **样式表现**: 
    *   当 `!isHovering` 时：`opacity: 0` (完全隐藏)。
    *   当 `isHovering` 时：`opacity: 1` 且 `scale: 1`。
    *   (用户反馈提到“在卡片外也能看到”，这主要是定位偏移和缩放逻辑导致的，Portal + Opacity 0 将彻底解决此问题)。

## 3. 实施步骤
1.  **添加依赖导入**: 在 `VibeCodingPage.tsx` 顶部导入 `createPortal` 和 `useMotionValue`, `motion`。
2.  **重构 `CardWrapper`**:
    *   移除 `cursorPos` state。
    *   初始化 `cursorX` 和 `cursorY` motion values。
    *   在 `mousemove` 监听器中直接调用 `.set()` 更新坐标。
    *   使用 `createPortal` 渲染光标元素。
    *   将光标的 `motion.div` 样式绑定到 motion values。
    *   确保 `onMouseEnter/Leave` 正确切换 `isHovering` 状态。
3.  **验证**:
    *   移动鼠标，确认光标是否精准跟随鼠标尖端。
    *   移出卡片，确认光标是否立即消失。
    *   确认 Hero 模式下光标完全不渲染。

## 4. 预期结果
*   光标精准跟随鼠标，无偏移。
*   仅当鼠标真正悬停在卡片区域内时，"VIEW" 光标才会出现。
*   鼠标移动不再导致页面掉帧。
