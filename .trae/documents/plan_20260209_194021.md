# Vibe Coding 页面溢出与裁剪逻辑修复计划

## 1. 问题分析
*   **现象**: `div` 出现切割（裁剪），原因是父级和子级容器都设置了 `overflow: hidden`。
*   **冲突点**:
    *   **Hero (ASCII) 模式**: 必须要有 `overflow: hidden`（或者说尺寸限制），以确保 ASCII 效果在变形过程中被正确裁剪，不会溢出到不该显示的地方。
    *   **Card (3D) 模式**: 需要 `overflow: visible`，因为 3D 卡片有阴影效果 (`box-shadow`) 和可能的 3D 旋转溢出，如果 hidden 会导致阴影或边缘被切掉。
    *   **状态切换**: 用户希望在 Hero 变 Card 的过程中能够保持视觉连贯性，但在 Card 状态下不能有切割。

## 2. 解决方案
采用动态 `overflow` 策略，根据当前模式 (`isHeroMode`) 动态切换样式。

### 2.1 外部容器 (`div` L513)
*   **修改**: 移除或放宽外部容器的 `overflow: hidden`。因为如果最外层裁切了，内部怎么改都没用。鉴于该容器是全屏 `100%`，通常不需要强制 hidden，除非有特定背景溢出。建议改为 `visible` 或移除该属性。

### 2.2 变形容器 (`motion.div` L522)
*   **修改**: 将静态的 `overflow: 'hidden'` 改为动态值。
    *   当 `isHeroMode === true` (Hero 模式) 时：保持 `overflow: 'hidden'`，确保 ASCII 视频被完美裁剪在圆角容器内。
    *   当 `isHeroMode === false` (Card 模式) 时：切换为 `overflow: 'visible'`，允许 3D 卡片的阴影和透视效果溢出容器边界。
    *   **注意**: Framer Motion 的 `layout` 动画可能会在属性切换瞬间产生视觉跳变，需要测试其平滑性。如果直接切换太生硬，可以考虑在 `onLayoutAnimationComplete` 回调中切换，或者接受 Hero 变形过程中短暂的 visible（但这可能导致 ASCII 溢出）。最佳实践是随状态直接切换。

## 3. 实施步骤
1.  **修改外部容器**: 在 `VibeCodingPage.tsx` 约 513 行，移除 `overflow: 'hidden'`。
2.  **修改变形容器**: 在 `VibeCodingPage.tsx` 约 522 行，将 `overflow: 'hidden'` 修改为 `overflow: isHeroMode ? 'hidden' : 'visible'`。
3.  **验证**:
    *   检查 Hero 模式下 ASCII 边缘是否整齐。
    *   检查 Card 模式下 3D 卡片阴影是否完整显示。
    *   检查从 Hero -> Card 过渡动画过程中是否有明显的视觉瑕疵。

## 4. 预期结果
*   Hero 模式：容器边缘清晰，内容被裁剪。
*   Card 模式：卡片悬浮，阴影完整，无切割感。
*   过渡：平滑自然。
