# Vibe Coding 页面布局与渲染修复计划

## 1. 核心目标
修复 Section 1 布局偏差，解决 ASCII 内容在状态切换时的尺寸适配问题，并移除不必要的硬编码高度以支持自适应卡片。

## 2. 详细修复方案

### 2.1 修复 Section 1 布局一致性
*   **问题**: 当前 Section 1 的布局结构虽然看似正确，但 `contentWrapper` 被移除导致 "VIBE CODING" 文字位置可能与原设计有细微偏差。
*   **方案**:
    *   恢复 `vibeCodingPageStyles.contentWrapper` 的部分结构逻辑。
    *   确保 "VIBE CODING" 文字在 Hero 模式下严格对齐到底部 15vh 区域，并且在垂直方向上绝对居中。
    *   调整 `motion.div` 的初始状态，确保首次加载时即为完美的 Hero 布局，无闪烁。

### 2.2 解决 ASCII 内容返回时不撑满问题
*   **问题**: `AsciiScene` 仅在挂载和窗口缩放时计算尺寸，无法感知父容器 `motion.div` 的动画变形。
*   **方案**: 在 `AsciiScene` 组件中引入 `ResizeObserver`。
    *   **监听目标**: 监听 `mountRef.current`（即 ASCII 场景的父 DOM 节点）。
    *   **回调逻辑**: 当检测到父容器尺寸变化时，实时调用 `handleResize` 更新 Three.js 的 Renderer 和 Camera 参数。
    *   **性能优化**: 在回调中加入 `requestAnimationFrame` 防抖，避免高频触发重绘。

### 2.3 优化卡片高度自适应逻辑
*   **问题**: `containerVariants` 中硬编码了 `height: '48.375vw'`，强制所有卡片为 16:9，可能限制了灵活性。
*   **方案**:
    *   移除 `containerVariants` 中的固定 `height`。
    *   利用 `aspect-ratio` CSS 属性来维持比例，或者允许 `Card3D` 内部内容撑开容器。
    *   考虑到 `motion.div` 需要在不同尺寸间平滑过渡，将使用 `aspectRatio: 16/9` 替代固定的 `vw` 高度计算，这样既保持了比例，又更符合现代 CSS 实践。

## 3. 实施步骤
1.  **修改 `AsciiScene` 组件**:
    *   引入 `ResizeObserver` API。
    *   替换原有的 `window.addEventListener('resize')` 逻辑（或与之共存）。
2.  **调整 `VibeCodingPage` 样式与结构**:
    *   微调 "VIBE CODING" 文字容器的 Flex 布局属性。
    *   更新 `containerVariants`，将 `height` 改为 `aspectRatio` 控制。
3.  **验证**:
    *   测试从 Section 2 滑动回 Section 1，确认 ASCII 画面是否瞬间铺满全屏。
    *   检查 Section 1 的文字布局是否与预期设计完全一致。

该方案将确保交互的流畅性与视觉的精确性。
