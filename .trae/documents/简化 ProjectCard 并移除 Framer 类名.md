## 可行性结论
可以简化 ProjectCard，并把所有 framer-* class 改成可读命名，但要做到 UI/交互一模一样，需要先把当前样式/交互映射成可读结构，逐步替换，否则会丢细节。

## 方案
1. **结构与样式梳理**
   - 解析 ProjectCard 现有 DOM 层级、关键 class 与 CSS 规则（含 hover、变体、绝对定位）
   - 输出“结构/样式/交互”映射表（哪些 class 对应哪些视觉块）

2. **建立语义化样式文件**
   - 新建可读 class（例如 `.card`, `.media`, `.arrow`, `.badge`）
   - 把 css4 中的对应规则迁移并整理到新样式文件

3. **重写 ProjectCard 组件**
   - 用语义化结构重写 JSX
   - 保留原交互逻辑（hover/动画/变体），确保行为一致

4. **逐项对齐验证**
   - 对比 hover 前后、箭头位置、视频/图片裁剪、文本排版
   - 有差异就回填修正

如果确认，我就开始做“结构映射 → 样式迁移 → 组件重写 → 像素级对齐”。
